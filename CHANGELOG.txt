KANT IDE changelog

2026-07-13 (3)
- Pin and eye icons in MAPPA are now roughly twice as large (40px normal, 60px enlarged for the drillable single pin), same drawn shapes, same proportional gap above the node.
- Fixed: clicking the eye icon did nothing. It sits on top of its node with no item-data of its own, so the view's click handler read the click back as "clicked empty canvas" and cleared every pin — hiding the eye — before Qt ever delivered the press to the eye's own handler. The view now recognizes a click landed on the eye specifically and lets it through untouched.

2026-07-13 (2)
- Fixed the real cause of "clicking a KANT tree section doesn't isolate the coding panel": for a legacy file with no #id marker, every reparse (and _open_file always reparses from disk) mints a fresh random uid, so the tree item's uid — captured at the last tree rebuild — could silently stop matching the freshly-opened tab, falling through to the whole-file view instead of the clicked element. Tree sections now carry their document order too and fall back to it when the uid lookup misses, the same recovery _navigate_to_element already used for this exact problem.

2026-07-13
- MAPPA drill-down redesigned per feedback: the drilled element is no longer a node in its own graph. It's fully detached and rendered as a fixed title card pinned to the view's top-right corner at a constant on-screen size (immune to zoom/pan) — the graph itself now shows only its direct children and their mutual references, nothing else.
- Added a layout direction toggle to the MAPPA toolbar: code flow reads left-to-right by default, click to flip the whole map (module rank and intra-cluster call-depth ranking) to right-to-left.

2026-07-12 (4)
- MAPPA: a node whose internal sub-elements have real cross-references between them (2+ children, at least one calling another) now shows an enlarged pin badge plus a drawn eye icon when it's the only pinned node; clicking the eye enters an internal-only "drill-down" view of just that element's own hierarchy, anchored top-left, with a way back to the full map.
- MAPPA: pin and eye badges are now drawn shapes (filled circle + number, drawn eye) instead of emoji glyphs, bigger and with a well-defined circular click/hover area, positioned with a real gap above the node instead of touching its edge.
- MAPPA: a colored reference arrow between a direct parent-child pair (e.g. a function referencing its own containing module) is no longer drawn — the neutral containment line already shows that relationship, so the reference arrow was pure redundancy.
- MAPPA: elements now cluster somewhat toward their common origin (same immediate KANT parent, or the same common-origin anchor) during layout, at a deliberately weak weight so the existing hierarchy/direction forces stay dominant. Node size, heatmap intensity, and the layout itself now also react to the connections tag filter, not just node visibility; added a "Appartenenza" toggle in the connections row for the neutral belonging connections themselves.
- The kant-code-map launch prompt (shown when opening a project with no KANT tags at all) is now one real dialog — provider, specific model, and reasoning effort together, with Cancel — instead of a plain 3-button choice. Effort is real for both CLIs (a direct `--effort` flag for Claude Code, a `-c model_reasoning_effort=` config override for Codex).
- Sending a chat message now silently scopes the AI's changes to whatever the coding panel is currently showing (the isolated element if one is filtered in, otherwise the whole open file) via the same hidden system-prompt channel already used for the KANT comment standard — never shown in the chat bubble. A new GLOBAL toggle at the top of the tree panel suppresses this scoping entirely for project-wide requests.
- Fixed: the title bar's bottom border separating it from the three panels below was never actually rendering — a bare QWidget doesn't paint stylesheet background/border without WA_StyledBackground, which was never set.
- Investigated a reported regression where clicking a KANT tree section stopped isolating the coding panel; traced and directly verified the isolation mechanism (_render_view) still works correctly and unchanged — could not reproduce a failure.

2026-07-12 (3)
- Added basic Python debugging: click a line's gutter to toggle a breakpoint, F5 launches the file under pdb with those breakpoints pre-armed. The terminal pane now forwards typed input to whatever process is running (not just Ctrl+C to kill), so stepping and inspecting variables at a "(Pdb)" prompt works directly in it. Fixed a real, currently-live bug found along the way: every terminal command crashed with AttributeError because TerminalPane referenced a `_error` slot that didn't exist.
- LSP diagnostics now override a merely "OK" local syntax check when a language server reports a real error — previously they were only appended as trailing text, never able to flip the status color.
- Project file reads are now cached by (mtime, size): re-scanning the project tree, xref, or KANT map no longer re-reads and re-parses every unchanged file's full text each time, only ones that actually changed.
- Found and fixed a real, compounding source of "dead space" in the KANT elements panel: every code block reserved ~14px for a horizontal scrollbar it almost never actually shows (ScrollBarAsNeeded's sizeHint was added unconditionally). Now only reserved when a line genuinely overflows and the scrollbar will actually appear.
- MAPPA hover popups (incoming/outgoing explanation) now wait a short moment before appearing instead of popping instantly, and now also work on hovering a node itself (its own incoming/outgoing), not just an edge.
- Added a collapse/restore tab for the AI terminal pane on the window's right edge, same tab-on-the-window-edge style and toggle behavior as the MAPPA tab; it remembers the pane's width and restores it exactly.

2026-07-12 (2)
- MAPPA: clicking a node now pins it instead of single-selecting — multiple nodes can be highlighted at once, each showing a small numbered "📌N" tag above it (N = pin order); clicking a pinned node again unpins just that one, clicking empty canvas clears every pin. Isolate/search still jump to a single node by replacing the pin set.
- MAPPA: every element now draws a plain black, arrowless line to its immediate containing element (module/class), separate from the colored directed reference arrows, so containment reads clearly alongside call direction. Fixed a real bug along the way: XrefMapDialog's display-node rebuild was dropping each element's container info even when it existed on the underlying element.
- MAPPA: added a second "Connessioni:" tag-toggle row directly below the node tag row, controlling which tags' reference edges are drawn at all (independent of whether the nodes themselves are shown) — e.g. show function nodes but hide every edge touching a function.
- The metadata editor now opens in one internal dialog (styled like MAPPA, not native) with all three fields together, directly on ⋮ click — no intermediate menu. The ⋮ button is bigger but height-capped so it no longer stretches each KANT element's header row.
- MAPPA's own scrollbars are gone (panning was already drag-based); the map's close-tab is now reparented onto the map's own window while it's open so a separate top-level dialog can never cover it and make it unclickable.
- The pinned edge popup's accent border is HOT (amber) instead of ACCENT (blue) — ACCENT is reused by every hover/button/selection in the app, so a pinned popup wasn't actually standing out.
- Fixed: selecting a whole file (or any tab with no section filter — the normal state right after opening one) always showed empty Incoming/Outgoing, because uid=None was treated as "nothing selected" instead of "the file's own top-level element is selected."
- The KANT-map filename + validation checkmark moved from below the project tree into the status bar, left of the UTF-8 indicator.
- KANT element headers: the name/short-description is now clearly bigger (12pt) than the extended description (9pt), not just 1pt apart.

2026-07-12
- MAPPA nodes never overlap: repulsion now accounts for each node's real (traffic-scaled) box size, the simulation runs in center coordinates internally so variable-sized boxes compare correctly, and a hard axis-aligned overlap-resolution pass cleans up whatever the soft force model still leaves touching. Node labels are elided to the node's actual pixel width via font metrics instead of a crude character-count guess.
- MAPPA layout now reads as logical call flow, not just file rank: each file's own elements are seeded left-to-right by intra-file call depth (source before target) instead of an arbitrary spiral, since the layout's seed-pull is strong enough that only the seed itself — not a competing per-iteration force — can reliably win out.
- Selecting a single element in MAPPA and then closing the map now opens that element in the coding panel underneath, instead of leaving whatever was open before.
- The coding panel is denser again: smaller code font, tighter CodeEdit padding/corner-radius and auto-resize allowance, and tighter FileTab margins/spacing, to fit more code on screen.
- The AI chat now shows an animated "· · ·" placeholder bubble the moment a prompt starts, replaced by the real streamed reply as soon as it arrives (and cleared, not left stuck, if a run ends with no output at all). Dropped the "# snapshot progetto creato" chat notification.
- The AI change review card's summary now matches a normal assistant chat bubble's styling (no separate icon avatar, same name-label/margins) instead of reading as a distinct panel.
- MAPPA: opens from a small tab stuck to the bottom-center window edge (toggles open/closed) instead of a button in the tree panel; a pinned edge popup now gets a visibly stronger accent border; the edge popup's size now tracks the camera's zoom level instead of staying fixed.
- Fixed a real source of extra vertical gaps between KANT elements: CollapsibleSection/LeafSection's nested content layout had no explicit spacing, so it fell back to the platform default (~6px) regardless of how tight the section's own margins were.
- MAPPA node box size and label font size now scale with code traffic (incoming+outgoing reference count): busy hubs render bigger and more legible, quiet leaves stay compact. Fixed a dead `node_font` that was built but never applied to labels along the way.
- KANT element widgets (CollapsibleSection/LeafSection, the coordinated-constants panel, the file view padding) are now much tighter — a second, more aggressive density pass on top of the earlier one.
- MAPPA: relayout animations no longer redraw every edge once per node per frame (O(nodes) redundant redraws) — exactly one redraw per animation frame, for visibly smoother motion at the same speed.
- MAPPA: node bubble size now follows the camera with damping (screen size ~ zoom**0.5, not zoom**1) — legible when zoomed out over the whole graph, not oversized when zoomed in close.
- MAPPA: nodes with no incoming/outgoing edges at all are anchored more strongly to their module's seed position, so they no longer drift far from their cluster with nothing but repulsion acting on them.
- The KANT element headers (CollapsibleSection/LeafSection) are more compact still, and the "Meta" text button is now a small ⋮ icon that opens a menu ("Modifica metadati…") instead of jumping straight to the editor.
- Fixed: selecting a module (or any element with nested KANT children) in the Incoming/Outgoing panel only showed that element's own direct references, not the union of everything its children reference; it now aggregates the whole subtree, excluding purely-internal references.
- MAPPA has a Heatmap toggle: colors nodes by connectivity (incoming+outgoing count) through a cold-to-hot gradient instead of by tag, to spot the busiest elements at a glance.
- The AI chat now has a per-agent, editable model picker (Claude Code: current Anthropic model IDs; Codex: best-effort recent model names, unverified) that adds `--model` to the launched command.
- AI change review is now an inline chat card (`ClaudePane.show_ai_review`/`_AiReviewCard`) instead of a separate modal dialog — the same summary/expand-into-detail UI, but embedded directly in the conversation and never blocking the rest of the IDE.
- The project tree is now boxed like the code panels (CODE_BG background, border, rounded corners) instead of blending into the white app background, and both the tree and the coding panel (fonts, padding, section spacing) are noticeably more compact to fit more on screen.
- MAPPA now opens fully expanded every time instead of collapsed, is noticeably larger by default, recomputes and smoothly animates node positions whenever the filtered set changes (tag toggle, file focus, isolate, expand/collapse, search, Riorganizza) instead of leaving survivors pinned to stale spots, and uses a geometric (not linear) cooling schedule for smoother settling.
- Fixed: the MAPPA edge popup only showed one aggregated reference per collapsed-module arrow even when several real references fed into it; it now lists every one.
- Fixed a Windows command-injection hole in the Run button (unquoted shell metacharacters in a file path), a bug where every save silently stripped a file's permissions, and a parser bug where a CATEGORY/tagline comment right before a CLOSED marker could be lost or reordered.
- AI-created symlinks are now stripped before every review/rollback (a snapshot is symlink-free by construction, so one appearing afterward is never safe to keep); an unresolved AI review snapshot is now persisted and offered for real recovery on the next launch instead of only being detectable, never restorable.
- MAPPA now follows directed data flow: cyclic modules are grouped, acyclic flow is ranked left-to-right, weighted barycenter ordering reduces crossings, and organic per-module clusters plus local forces keep modules apart. Edge hover shows green incoming and red outgoing references; clicking pins the popup.
- Initial KANT tagging/map generation now grants automatic permissions only for that confirmed AI operation (Claude permission bridge or Codex `--full-auto`), then restores normal permission handling. AI change review now uses assistant cards and rounded chat-style inspection surfaces.
- Restored reliable module expansion/collapse in MAPPA and stabilized its camera: zoom and center now survive expansion, tag filtering, background graph refresh, and node drag; the graph recenters only for explicit framing actions.
- Replaced MAPPA's rigid file columns with an Obsidian-style deterministic force layout. Nodes are drag-and-drop movable, curved arrows follow them live, positions persist per project, and `Riorganizza` resets to the automatic layout.
- Added interactive Claude permissions inside the AI chat: deny, allow once, allow by tool for the current session, plus an explicit Automatic mode. Requests use an authenticated local MCP bridge; project snapshots and final selective review remain active in every mode.
- Replaced the AI keep/rollback prompt with a Codex-style change-review dialog: compact file and line totals, expandable file list, per-file/per-hunk inspection, editable final content, selective accept/reject, accept-all, and full rollback.
- Moved the terminal to a full-width bottom panel beneath the tree and editor, ending before the right-side AI pane.
- Restyled the AI terminal as a messaging chat with right-aligned user bubbles and streamed left-aligned Claude/Codex replies.
- Reduced `MainWindow` from 2,986 to 2,317 lines without changing the UI: themed dialogs moved to `kant/dialogs.py`, pure scans/search/map/validation to `kant/projectops.py`, and watchers/conflicts/AI snapshot/file lifecycle to `kant/workspace.py`.
- Atomic writes now retry short-lived Windows file locks caused by concurrent background readers.

2026-07-10
- Incoming/Outgoing are now navigable lists: Incoming shows what references the selected element and from which file (`←`), Outgoing what it references and to which file (`→`); rows are labeled by the element's short description (as in the left tree) with its category text on hover; double-click a row to jump straight to that element.
- MAPPA now opens the cross-reference graph in a frameless dialog internal to the IDE (floating over the editor, not a separate OS window). Modules start collapsed — the graph opens as a clean file-to-file overview and you double-click a module to expand it into its elements — and tests are hidden by default. Nodes are labeled by short description with category text on hover. Navigation aids: name/description search that expands+focuses a hidden match, per-tag toggles (also the color legend), a file selector isolating one file plus neighbors, isolate-selected, expand/collapse-all, zoom/fit, and double-click-a-leaf to jump to it in the editor. Selecting a node dims everything except it and its neighbors.
- BREAKING: dropped hand-written `[TAG INCOMING]`/`[TAG OUTGOING]` comments from the KANT convention. New deterministic cross-reference system (`kant/xref.py`, zero dependencies): every KANT-tagged element's code is tokenized and identifier matches against other elements' names become directed reference edges. The Incoming/Outgoing panels now show who references the selected element and what it references (any tag, project-wide); a new MAPPA tab draws the whole reference graph — file columns ordered callers-first, nodes colored by tag, wheel zoom, drag pan, click a node to highlight its edges. Old files with INCOMING/OUTGOING comments still parse and round-trip losslessly, but nothing reads or writes them anymore.
- BREAKING: OPEN/CLOSED markers now carry a unique `#id` (`[TAG OPEN #id] Name` / `[TAG CLOSED #id] Name`). parse_kant enforces strict LIFO nesting and id matching and raises KantParseError (with line numbers) on a mismatch instead of silently guessing; check_file_syntax surfaces the same errors, plus file-wide #id-uniqueness, through the existing syntax status bar.
- Legacy (id-less) files migrate automatically: opening one assigns fresh ids in memory, and the next save writes them back into the source. No batch migration needed.
- The IDE now owns KANT_<project>.md as generated output: it resyncs on every save and offers to generate one on project open if none exists, instead of only detecting whether one is present.
- KantHighlighter now highlights the `[TAG OPEN/CLOSED #id]` marker bracket distinctly from surrounding comment text, for the rare case a marker leaks into a code block unparsed.
